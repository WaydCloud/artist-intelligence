# AGENTS — 에이전트 작업 규칙 (하드 게이트)

> 이 파일은 `CLAUDE.md`로 임포트되어 매 작업에 자동 로드된다. 위반은 리뷰/CI에서 차단한다.
> 규칙은 "정답 강제"가 아니라 **핵심 흐름을 지키기 위한 최소 계약**이다.

## 0. 최우선 불변식 (깨면 머지 불가)

- **핵심 흐름**(`모듈 CLI → 스키마 유효 report.json → 대시보드 렌더`)을 깨는 변경 금지.
- `packages/report-schema/report.schema.json`을 어기는 출력/변경 금지. 스키마 변경은 **별도 승인 + 대시보드 동시 갱신**이 전제.

## 1. 구조 · 중복 방지

- **같은 기능/유틸을 중복 생성하지 않는다.** 새로 만들기 전 기존 `modules/`·`packages/` 검색.
- **승인 전 대규모 구조 변경 금지** (폴더 이동, 스택 교체, 공통 규격 변경).
- 새 의존성은 **근거 + 승인**. 무거운 모노레포 툴(Nx/Turbo) 도입 금지.

## 2. 문서 우선 (하네스 준수)

- 기능 추가/변경 전 해당 모듈 `SPEC.md`·`RULES.md` 확인. 규칙에 없으면 먼저 `RULES`에 정의하거나 질문한다.
- 도메인 판단은 `DOMAIN.md`에 근거. 근거가 없으면 **추측하지 말고 질문**.
- 결정/규칙이 바뀌면 **문서를 먼저 갱신**하고 코드에 반영.

## 2.1 기준 원장 (Criteria Ledger)

> 미시 판단(측정·분류)의 기준은 임의가 아니다. **책임소재 불변식**([`DOMAIN.md`](DOMAIN.md) §0)의 증거 레이어를 지키는 규율.

- 모듈이 산출하는 **지표·임계값·분류 기준**은 해당 `RULES.md`에 **원장 형식**으로 명시한다: `정의 · 도메인 근거 · 하중여부(튜닝가능) · 테스트(TESTS.md) · 한계/불확실성`.
- **소유 분리**: 엔지니어는 기준의 **형식**(정의·근거·테스트·불확실성·튜닝 가능성)을 소유하고, 기준의 **값(임계값)**은 도메인 소유자(A&R·안무감독)가 보고·반박·재조정할 수 있게 **노출**한다. 임계값을 코드에 은닉하지 않는다.
- **트리아지**: 임계값이 인간의 결정을 바꾸는 **하중받는 기준**만 도메인 근거·테스트·튜닝 노출을 요구한다. 관습적 기준은 상식적 기본값을 문서화하고 넘어간다.
- 기준은 **버전 매겨진 가설**이다(진리 아님). 재조정이 싸고 추적 가능하게 둔다. **케이스에 오버핏 금지** — 기준은 도메인 근거에서 정한 뒤 결과를 본다.
- 원장의 원형: [`modules/chart-history/RULES.md`](modules/chart-history/RULES.md) §3 지표 정의.

## 3. 정적 우선

- 임의로 라이브 백엔드·서버·인증·DB 추가 금지(`ARCHITECTURE.md` 위반). 필요하면 제안 후 승인.

## 4. 품질 게이트 (커밋 전 통과)

- `lint` · `typecheck` · 해당 모듈 `smoke`(유효 `report.json` 산출) · `schema-validate`.
- **실패를 우회(스킵/무시)하지 않는다.** 원인을 수정한다.

## 5. 윤리 · 과대주장 금지

> 아래 금지들은 **책임소재 불변식**([`DOMAIN.md`](DOMAIN.md) §0)의 결과다: 판단은 책임질 인간에게, 도구는 증거만. 평결(단정)은 결정 레이어 침범이라 금지된다.

- ❌ "AI가 히트/스타를 예측한다" 단정 · "이 연습생은 데뷔감/아니다" 단정 · 다이내믹 점수 = 춤 실력 총점 등치.
- ✅ "이 축에서 이런 신호가 있다 — 참고하라." 불확실성·표본·한계 병기.
- 미성년·초상·저작권·개인정보 준수. **원본 저작물 재배포 금지**(파생 분석만).

## 6. 불확실하면 질문

- 추측으로 진행하지 않는다. 핵심 흐름·규격·도메인에 의문이 있으면 멈추고 묻는다.

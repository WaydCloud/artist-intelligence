# DESIGN — 클라이언트 대면 하네스 (브랜드·UI 규격)

> **클라이언트에 접근하는 모든 코드(페이지·컴포넌트·문구)는 이 문서를 거친다.** 위반은 리뷰에서 차단([`AGENTS.md`](AGENTS.md) §7).
> 아티스트 "브랜드" 분석([`docs/02-trend-brand.md`](docs/02-trend-brand.md)·[`docs/03-brand-harness.md`](docs/03-brand-harness.md))과는 무관한 문서다 — 여기서 브랜드는 **우리 서비스의 브랜드**를 뜻한다.

## 0. 정체성

- **브랜드명(대외 표기): `WaydCloud`** — 워드마크는 대문자 `WAYDCLOUD`, 디스플레이 서체 + 넓은 자간.
- 제품: Artist Intelligence (리포트 대시보드), Utilities (보조 도구).
- 미학 키워드 6: **시네마틱 · 미니멀리즘 · FUTURA · PRETENDARD · 에디토리얼 · 글래스모피즘 · 메탈릭**. 아래 §1이 키워드를 집행 가능한 규칙으로 번역한다.

## 1. 미학 원칙 (키워드 → 규칙)

| 키워드 | 규칙 |
|---|---|
| 시네마틱 | 넓은 여백 + 큰 명암 대비 + 배경은 은은한 라디얼 글로우/비네트/필름 그레인(토큰 `--glow`·`--vignette`·`--grain`)까지만. 그레인은 저불투명 오버레이(`--grain-opacity`, ≤ 0.08) — 텍스트 대비를 깎으면 뺀다. 사진·영상 배경 금지(정적 우선). |
| 미니멀리즘 | 표면당 시각적 강조 1개. 장식은 기능(위계·상태)에 종속. 요소를 더하기 전에 뺄 것을 먼저 찾는다. |
| 에디토리얼 | 구조는 타이포 위계로 전달(크기·굵기·자간). 본문 줄길이 ≤ 68ch. 그리드 정렬 이탈 금지. |
| 글래스모피즘 | **표면 위계 표현 전용**(카드·네비). 토큰 `--glass-bg`·`--glass-border` + blur만 사용. 차트·표·본문 텍스트 뒤에는 금지(§4). |
| 메탈릭 | **포인트 액센트 전용**(워드마크·제목·아이콘·경계 하이라이트). 토큰 `--metal` 그라디언트만. 대면적 배경 금지. |

## 2. 타이포그래피

- **디스플레이(워드마크·대제목)**: Futura → 폴백 **Jost**(무료 기하학적 산세리프, next/font 셀프호스트). Futura는 상용 라이선스 — 웹폰트 라이선스를 확보하기 전까지 웹에서는 Jost가 실제 렌더 서체다(스택에 Futura를 앞세워 로컬 설치 환경만 Futura로 렌더).
- **본문·UI·한글**: **Pretendard**(Variable, CDN).
- 스택 정본은 `apps/dashboard/app/globals.css`의 `--font-display`·`--font-body`. 컴포넌트에서 폰트명 직접 지정 금지.

## 3. 토큰 (단일 정본)

- **정본: [`apps/dashboard/app/globals.css`](apps/dashboard/app/globals.css) CSS 변수.** 컴포넌트에는 `var(--…)` 참조만 — hex·rgb 하드코딩 금지.
- 기존 **dataviz 토큰**(`--surface`·`--ink`·`--series`… )은 검증된 팔레트다. 브랜드 작업이 이 값을 임의 변경하지 않는다(변경은 대비 검증 후 별도 승인).
- **브랜드 토큰**(추가분): `--font-display` `--font-body` `--glass-bg` `--glass-border` `--metal` `--glow` `--vignette` `--grain` `--grain-opacity`. 라이트/다크 각각 정의.
- 브랜드 표면의 배경 레이어(글로우·비네트·그레인)는 `components/BrandBackdrop.tsx` 하나로만 렌더 — 페이지마다 재구현 금지.

## 4. 표면 위계 (어디에 무엇을 쓰나)

| 표면 | 범위 | 허용 |
|---|---|---|
| **브랜드 표면** | 랜딩, 네비/셸, 마케팅성 페이지 | 시네마틱 배경 + 글래스 + 메탈릭 + 디스플레이 서체 |
| **데이터 표면** | 대시보드, 차트, 리포트 본문 | dataviz 토큰만. **글래스·메탈릭·글로우 침범 금지** — 증거 가독성이 브랜드보다 우선(책임소재 불변식의 연장) |

## 5. 모션

- 마이크로 인터랙션 150–300ms `ease-out`, 대형 전환 ≤ 400ms. 무한 반복 애니메이션 금지.
- `prefers-reduced-motion` 존중(전환 제거 또는 즉시 완료).

## 6. 접근성·문구 불변식

- 라이트/다크 **둘 다** 성립해야 머지 가능. 본문 대비 WCAG AA(4.5:1) 이상.
- 글래스 표면 위 텍스트가 대비를 확보하지 못하면 글래스를 포기한다(반대가 아니라).
- UI 카피도 [`AGENTS.md`](AGENTS.md) §5를 따른다: 단정·과대주장 금지, 신호·참고 어법.

### 6.1 카피 스타일 (클라이언트 대면 문구 한정 — 내부 문서 제외)

> 적용 대상엔 **모듈이 생성하는 표시 문구**(report.json의 `title`·`insights`·`recommendations`·`note`)도 포함된다 — 대시보드에 그대로 렌더되기 때문.
> 예외: **기계 파싱 규약**(예: 프로필 라인의 ` — ` 구분자, signal-bridge RULES §4.1)은 데이터 형식이지 카피가 아니다. 규약 변경은 파서·RULES 동시 갱신이 전제.
> UI에 내부 문서 참조(`§0`·`D-016` 등)를 노출하지 않는다 — 근거는 문서에, 화면엔 결론만.

- **1차 해석 원칙**: 읽자마자 이해되는 문장만. 은유·중의·수사로 2차 해석을 요구하지 않는다.
- **em dash(—) 금지.** 필요하면 문장을 나누거나 쉼표·마침표로.
- **AI 상투 표현 금지**: "혁신적인"·"완벽한"·"~의 세계로" 류 과장 수식, 기계적 나열투, 불필요한 느낌표.
- **전문용어는 일상어 우선.** 불가피하면 첫 노출에 짧은 풀이 병기([`docs/03-brand-harness.md`](docs/03-brand-harness.md) 용어 번역 규율과 동일 방향).
- **명사형 종결 우선**: "준비 중"(○) "준비 중입니다"(✕). 행동 안내 등 서술형이 더 명확한 경우만 예외.
- **효율**: 한 문장 한 정보. 짧게 쓸 수 있으면 짧게.

- [ ] 문구에 단정·과대주장 없음, 카피 스타일 준수(§6.1)

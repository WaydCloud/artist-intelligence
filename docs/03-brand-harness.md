# 03 · 브랜드 하니스 — 기획 원칙 학습 에이전트 (추론 레이어)

> **기획 추론** 단계. [`02-trend-brand.md`](02-trend-brand.md)의 데이터 위에서 *판단*을 돕는다.

## 용어 정리 (중요)

"Harness / Agent Skills / Deep Search"는 **개발자 용어**다. 면접관에겐 안 통한다. 포트폴리오 문서·발표에선 이렇게 번역:

- ~~Harness~~ → **"기획 원칙을 학습한 의사결정 지원 에이전트"**
- ~~Agent Skills + Deep Search~~ → **"기획서 기반 검색·분석 자동화"** (내부적으로 RAG + 에이전트형 리서치)

코드/설계 문서에선 원 용어 써도 되지만, 대외 서술은 위로.

---

## 이게 하는 일

1. **기획 원칙 흡수(RAG)** — 그룹의 기획서·컨셉 정의·브랜드 가이드(세계관, 타깃, 톤&매너, 레퍼런스)를 벡터화해 "우리 팀의 방향성"을 에이전트가 참조.
2. **현황 대조** — 최근 활동/콘텐츠/지표가 그 원칙과 **일관되는가, 이탈(drift)했는가**를 점검.
3. **유사 사례 리서치** — 유사 그룹의 브랜드평판이 언제·어떻게 올랐는지 웹/데이터에서 근거 수집.
4. **근거 있는 제안** — "이렇게 해라(정답)"가 아니라 **"이런 옵션들이 있고 각각의 근거는 이렇다"**.

---

## 아키텍처

```
기획서/브랜드 가이드 ──▶ [임베딩·인덱스] ──┐
                                          ├──▶ 에이전트(추론) ──▶ 리포트
현황 데이터(02 모듈) ─────────────────────┤          │
웹/데이터 리서치(툴 호출) ────────────────┘   [일관성 점검 · 옵션 제시 · 인용]
```

- **RAG**: 기획 문서 인덱싱(임베딩 + 벡터 검색).
- **에이전트 루프**: 리서치 툴(웹 검색/데이터 조회)을 호출하며 근거 수집 → 인용과 함께 결론.
- **출력**: 공유 리포트 규격. `insights`=일관성/이탈 진단, `recommendations`=근거·인용 붙은 옵션.

---

## 포지셔닝 (순진해 보이지 않으려면)

- **정답 기계가 아니라 옵션 제시기.** "추천 행동"이 외부인이 가장 순진해 보이는 지점. 항상 근거·인용·불확실성을 함께.
- **일관성 감사(consistency audit)** 프레이밍이 강력: "이 콘텐츠가 우리가 정한 세계관/톤과 맞는가?"를 자동 점검하는 도구는 기획팀에 실제로 유용.
- **사람이 최종 결정.** 에이전트는 근거를 모아 테이블에 올릴 뿐.

---

## 빌드 플랜

- **v1**: 기획서 1건 인덱싱 + "이 신곡 컨셉/티저가 우리 방향성과 일관적인가" 일관성 리포트 1개.
- **v2**: 02 모듈 데이터 연결 → 유사 그룹 벤치마킹 자동 리서치 + 근거 인용.
- **v3**: 대시보드에서 질의 → 근거 붙은 옵션 카드 출력.

## 주의

- 인용 없는 단정 금지(환각 리스크). 모든 제안에 출처.
- 기획서는 민감 자산 — 데모는 가상 그룹의 가상 기획서로.

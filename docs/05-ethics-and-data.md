# 05 · 윤리 · 데이터 원칙 (거버넌스)

> 이 문서 자체가 포트폴리오 자산이다. **성숙도**를 보여주고, 순진한 실수를 막는다. 채용자에게 "이 사람은 리스크를 안다"는 강한 신호.

## 1. 미성년 · 생체 데이터

- 트레이니는 **미성년자**인 경우가 많고, 안무 영상은 **생체(신체) 데이터**다.
- 원칙: **동의(consent) 우선**, 목적 한정, 최소 수집, 보관 최소화.
- 포트폴리오 데모는 **공개 영상 / 본인 영상 / 합성·가상 데이터**로 한정. 실제 트레이닝 평가에 쓰려면 별도의 동의·법률 검토 전제.

## 2. 증폭 원칙 (대체 아님)

- 모든 모듈은 **의사결정 지원**이다. 캐스팅·선곡·데뷔를 자동 결정하지 않는다.
- 리포트는 "정답"이 아니라 **근거 있는 옵션/피드백**. 최종 판단은 사람.
- UI 문구·리포트 톤에서 이 원칙을 일관되게 유지(과대 주장 금지).

## 3. 편향(bias)

- **포즈 추정**은 체형·의상·조명·피부톤에 따라 정확도 편차 → 다이내믹 지표에 영향. 정규화·한계 명시.
- **아티스트 지문/브랜드평판**은 과거 성공 사례에 편향(생존편향). "예측"이 아니라 "참고"로.
- 문화·미(美) 기준의 주관성 인정. 비주얼/스타성 같은 축은 **정량화하지 않는다**고 명시.

## 4. 저작권 · 데이터 출처

- MV·안무·음원은 저작물. **파생 분석(지표·히트맵)만 산출**하고 **원본을 재배포하지 않는다.**
- 대용량 원본은 `.gitignore` — 저장소엔 파생 결과(JSON·작은 이미지)만 커밋.
- 스크래핑은 각 사이트 약관·robots 준수, 공개 집계 우선.
- 각 모듈 README에 **데이터 출처·라이선스·수집 방법** 명시.

## 5. 투명성 · 겸손

- 지표에 **불확실성·표본 크기·한계**를 함께 표기.
- 인과 주장은 가설로 표기(상관 ≠ 인과).
- 에이전트(브랜드 하니스) 출력은 **인용 필수** — 근거 없는 단정 금지.

## 6. 과대 주장 금지 목록 (하지 말 것)

- ❌ "우리 AI가 다음 히트/스타를 예측한다"
- ❌ "이 연습생은 데뷔감/아니다"를 단정
- ❌ 다이내믹 점수를 "춤 실력 총점"으로 등치
- ✅ "이 축(다이내믹/적합도)에서 이런 신호가 있다. 참고하라."

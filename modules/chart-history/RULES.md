# chart-history · RULES (불변식 · 규칙)

> 지표를 **정확히** 정의해 즉흥 산출을 막는다. 이 정의를 어기는 산출은 금지.
> 값 무결성은 [`TESTS.md`](TESTS.md)에서 검증한다.

## 1. 소스 · 수집 규칙

- 소스 3레일(v4·D-016, 전부 무료·저ToS): ① **Kworb Spotify** 정적 차트 테이블(예 `spotify/country/kr_daily.html`, D-005) ② **Apple 공식 RSS**(`rss.marketingtools.apple.com` most-played — 공식 공개 피드, `collect-apple`이 동일 스토어 계약(facts-only 테이블+메타)으로 변환 저장) ③ **Kworb YouTube**(`youtube/insights/<cc>_daily.html`).
- **수집과 분석 분리**: `fetch`/`collect`(라이브·네트워크) ↔ `analyze`/`signals`(오프라인·결정적, 스모크 경로). 스모크는 네트워크를 타지 않는다.
- **사실 필드만 저장**: 순위·아티스트/곡·차트수명·스트림 수치. **사이트 크롬·광고·저작 콘텐츠 미저장**(픽스처는 `<table>`만 + 메타 주석). 메타에 `platform:`(spotify|apple|youtube, 없으면 spotify=구 스냅샷 호환).
- 차트 순위·수치는 **사실**이며 저작물이 아니다. 그러나 Kworb는 **집계값**(플랫폼 공식 지표와 다를 수 있음) → 리포트에 명시(§4).
- 저빈도 접근, `User-Agent` 명시. 대량·고빈도 크롤 금지.
- **플랫폼 혼합 최고순위**(D-016, signals): chart-rank 시리즈 값 = 일자별 **전 시장·전 플랫폼 최저 rank**. *도메인 근거*: 브리지 온셋 질문("차트에 진입했는가")은 플랫폼 불문 최초 진입이 답 — 플랫폼별 수용층이 달라 단일 렌즈는 사각(Apple/YouTube에서만 잡히는 팀 실측). *한계*: top-N 천장·규모가 플랫폼마다 달라 **rank 절대값의 플랫폼 간 비교 금지** — provenance note로 전파, act별 `platforms` 맵으로 어느 렌즈에 잡혔는지 노출. 스토어 레이아웃 `chart/<platform>/<cc>/`.
- **렌즈 온셋**(D-016 ②, signals): act별 `platformOnsets` = 렌즈별 **첫 관측일** + provenance `platformFirstDates`(렌즈별 수집 첫날). *한계(좌측 절단)*: 수집 첫날과 같은 온셋은 그 전부터 차트인했을 수 있음 — 소비자(브리지)는 **수집 개시 이후 온셋만** 시차 집계에 사용해야 한다(검열 보정). 시차 = 시간 순서, 인과 아님(§0).
- **3렌즈 통합 진입 지도**(D-016 ①, analyze 수평 뷰): 로스터 act × 시장, 셀 = **어느 플랫폼이든** 최고순위. *도메인 근거*: 화이트스페이스 질문("어느 시장에 아직 없는가")의 정답은 플랫폼 불문 진입 — 빈칸 = 3렌즈 전부 미진입(진짜 공백). 숫자는 참고(렌즈 간 비교 금지). spotify-only 화이트스페이스와 병존(최심 레일 심층 vs 통합 공백).

## 2. 컬럼 매핑 (Kworb daily)

| idx | 컬럼 | 의미 | 파싱 |
|---|---|---|---|
| 0 | Pos | 현재 순위 | int |
| 1 | P+ | 순위 변동 | `=`→0, `+N/-N`→부호정수, `NEW/RE`→None |
| 2 | Artist and Title | "Artist - Title" | 첫 ` - `로 분리 |
| 3 | Days | 차트인 일수 | int |
| 4 | Pk | 최고 순위 | int |
| 6 | Streams | 일간 스트림 | 콤마 제거 int |
| 7 | Streams+ | 일간 증감 | 부호정수 |
| 10 | Total | 누적 스트림 | int |

- **분리 한계**: 곡 제목에 ` - ` 가 포함되면 오분리 가능 → 알려진 한계로 수용(v2에서 링크 기반 개선 여지).

## 3. 지표 (정의)

> 이 절이 이 모듈의 **기준 원장**([`../../AGENTS.md`](../../AGENTS.md) §2.1) — 각 지표는 정의를 명시하고, 하중받는 기준(임계값 有)은 도메인 근거·테스트([`TESTS.md`](TESTS.md))·한계를 갖는다. 임계값은 코드에 은닉하지 않는다(예: `최고 상승`의 `P+ > 0`).

- **차트 진입 곡** = 파싱된 유효 엔트리 수.
- **고유 아티스트** = distinct `artist`.
- **1위** = `rank == 1` 엔트리(스트림·누적 hint).
- **최장 차트인** = `max(Days)`.
- **최고 상승** = `max(P+)` 중 P+ > 0 (없으면 지표 생략).
- **Top 10 · 일간 스트림** = `streams` 내림차순 10.
- **Top 아티스트** = 아티스트별 `Σ streams` 내림차순 10.

## 4. 출력 규칙 (report.json)

- `moduleId = "chart-history"`. 필수 상위 필드는 공유 스키마 준수([`../../packages/report-schema/report.schema.json`](../../packages/report-schema/report.schema.json)).
- `charts[].data` 규약: bar = `[{ "name": str, "value": number }]` (대시보드 렌더 계약).
- `generatedAt` = 실행 시각(UTC ISO). 지표·차트는 입력에서 **결정적**으로 도출(시각 외 비결정 금지).
- **엔트리 0**: 크래시 금지. 유효 report를 내되 `insights`에 "엔트리 없음"을 명시(누락 금지).
- `insights`에 **반드시 병기**: (a) 단일 스냅샷 = 시계열 아님, (b) Kworb 집계값 한계.

### 4.1 v2 · 다중 스냅샷 (국가 교차 / 날짜 시계열)

- `analyze`에 스냅샷을 **여러 개** 주면 v2. **기준(base) = 첫 스냅샷**(전체 v1 리포트) + 교차 섹션.
- **차원 자동 감지**: 모든 스냅샷에 `country`가 있고 서로 다르면 → **국가 차원(heatmap)**. 아니면 서로 다른 `snapshot_date`가 2개 이상이면 → **날짜 차원(line)**. 둘 다 아니면 기준만 사용(insights에 명시).
- **매칭 키**: `artist|title`(소문자·trim). 표기·romanization·`feat`·`- Remastered` 차이로 **오분리/누락 가능** → 한계 insight 필수(v3 엔티티 조인으로 해소).
- **`charts[].data` 규약** (대시보드 렌더 계약):
  - `heatmap` = `{ "rows": [track], "cols": [market], "cells": [[rank|null]] }`
  - `line` = `{ "x": [label], "series": [{ "name": track, "values": [rank|null] }] }`
- **교차 지표 정의**: 교차 시장 = 스냅샷 수 · 전 시장 진입 = base Top12 중 전 시장 charting 곡 수 · 최광역 곡 = base Top12 중 최다 시장 진입 · (날짜) 기간 최고 상승 = 첫→끝 순위 개선폭 최대.

### 4.2 v3 · 엔티티 마스터 조인 (MusicBrainz + Wikidata)

- **소스**: **MusicBrainz 1차**(CC0, `artist:"name"`, `score < min(기본 90)`면 미해석) → **Wikidata 폴백**(v3.1, 다국어 라벨·별칭 검색으로 로마자 해소; `wbsearchentities`→`wbgetentities`로 P495/P27 국가, P297로 ISO 코드). 각 레코드에 `source` 기록. 추측 금지 — 국가 없으면 미해석.
- **v3.6 데뷔연도·에이전시**(코호트 근거): `debut`(Wikidata **P571** 결성연도 ≈ 데뷔 시기) · `agency`(**P264** record label ≈ 소속사). **정확성 규율**: 이름 검색은 동명이인 오매칭 위험(예: 'Jimin'→1955 오류)이라, **MB의 wikidata 링크(url-rels, 검증된 QID)에서만** 데뷔·에이전시를 가져온다(`wd_id` 저장). MB 링크 없으면 미해석(정확성 > 커버리지). country 해석 **무회귀**(별도 경로). WD 폴라이트 sleep으로 레이트리밋 회피.
- **수집/조인 분리**: `enrich`(라이브·rate-limit 1.1s, UA 명시; `--no-wiki`로 폴백 off) → **커밋 가능한 엔티티 맵**(공유 레이어 [`../../packages/entity-master/entities.json`](../../packages/entity-master/entities.json), CC0 사실만) → `analyze --entities`가 **오프라인**으로 조인(결정적).
- **매칭 키**: `normalize.primary_artist`(협업 마커 이전의 대표 아티스트). 정규화(`canonical_key`)는 v2 교차 매칭에도 적용해 표기 차이에 견고.
- **지표/차트**: 국내(KR) 아티스트 = 해석된 상위 아티스트 중 `country==KR` 비율(%) · 원산지 분포 bar. insight에 **해석 커버리지 + 소스 출처(MB·Wikidata)** 병기.
- **한계 필수 명시**: (a) 해석 커버리지(`resolved/total`), (b) 스타일라이즈드·혼종 표기로 **여전히 미해석되는 아티스트 존재** → insights에. 미해석은 원산지 집계에서 제외(0으로 왜곡 금지).
- `country`/`type`은 커뮤니티 편집 기반 → **불완전·오류 가능**, "참고" 성격.

### 4.3 v3.2 · 헤더 파서 · 크로스뷰 모멘텀 · 축적

- **헤더 기반 파싱**: 컬럼을 **헤더 라벨**로 매핑(고정 인덱스 아님) → daily(`Days`)·weekly(`Wks`) 등 레이아웃 변형에 견고. 라벨 없으면 해당 필드 `None`.
- **뷰 차원(momentum)**: 스냅샷들이 **같은 국가·같은 날짜, 다른 chart 라벨**이면 뷰 차원. base(첫=일간)의 Top20을 상대(주간)와 매칭 → **모멘텀 = 주간순위 − 일간순위**(+ = 일간이 앞섬 = 상승세). 지표 `상승세 곡`/`최고 모멘텀`, bar 차트.
  - **차원 우선순위**: 국가(distinct country) → 날짜(distinct date) → 뷰(distinct chart). 하나만 적용.
  - 모멘텀은 **가속 신호(참고)** — "히트 예측"이 아니다(§5).
- **축적 파이프라인**: `collect`(라이브)가 스냅샷을 **`data/snapshots/<store>/<date>.html`** 로 적재(날짜별, `.gitignore` 대상). `analyze <store-dir>`는 디렉터리의 `*.html`을 날짜순으로 읽어 **날짜 라인 시계열** 산출. 실데이터 라인은 스냅샷이 **여러 날 쌓여야** 의미 생김(단일 날짜면 1점).

### 4.4 v3.3 · 크로스소스 positioning (Spotify × Apple)

- **소스 차원**: chart 라벨에서 **서비스**(Spotify/Apple/Melon)를 도출. 스냅샷들이 **같은 국가, 다른 서비스**면 크로스소스. 차원 우선순위: 국가 → **소스** → 날짜 → 뷰.
- **매칭 = 제목(`canonical_title`) 기준 + 엔티티 콜리전 가드**(v3.4): Apple KR은 아티스트가 한글("코르티스")·Spotify는 Latin("CORTIS")이라 아티스트 문자열로는 조인 불가. 제목은 언어중립인 경우가 많아(REDRED=REDRED) 제목으로 매칭하되, **양쪽 아티스트가 서로 다른 엔티티로 확실히 해석되면 콜리전으로 거부**. 아티스트 엔티티는 **MB 별칭**(코르티스↔CORTIS, `enrich`가 `inc=aliases`로 수집 → `alias_index`)으로 크로스언어 인식. → **커버리지 무회귀 + 콜리전 방지 + 엔티티 확인 카운트**.
- **지표/차트**: 양대 플랫폼 진입 = base Top20 중 상대에도 진입 · 엔티티 확인 = 양쪽이 같은 엔티티로 해석된 곡 · 플랫폼 편중 = `상대순위 − base순위` 최대 · heatmap(곡 × [Spotify, Apple]).
- **한계 필수 명시**: **제목 언어가 갈리는 곡("만찬가"↔"Bansanka")은 미매칭**. 근본 해결(레코딩 MBID)은 **실측 결과 MB 레코딩 커버리지 부재로 불발**(신곡·크로스언어 레코딩 없음) → **유료 ISRC 애그리게이터**가 다음 과제. positioning은 플랫폼 이용행태 신호(참고)일 뿐 인기·품질 단정 금지(§5).

### 4.5 v3.5 · 다국가 지리 리프레임 (앵커-중립 · 송곳 타겟팅 증거)

> **왜**: §4.1의 크로스컨트리 heatmap은 **앵커 편향**이다 — row가 기준(첫) 국가의 Top12뿐이라 "타국에선 큰데 앵커에선 낮은 곡"을 **숨긴다**. 글로벌-우선 기획(특정 다수 국가를 송곳 타겟팅)엔 **앵커-중립** 지리 증거가 필요하다. 이 절은 그 프레임의 **기준 원장**([`../../AGENTS.md`](../../AGENTS.md) §2.1).

- **트리거**: 크로스컨트리(§4.1)에서 **조사 국가 수 ≥ `N_geo`(기본 6)**. 미만이면 기존 앵커 Top12 뷰만(하위호환).
- **지역 그룹 (region map)**: ISO alpha-2 → 권역. **엔지니어가 형식 소유 · 그룹핑은 튜닝 가능**(도메인 소유자 재조정). 기본: 동아시아·동남아·남아시아·유럽·북미·중남미·MENA·아프리카·오세아니아·기타. `GLOBAL`은 국가 아닌 **세계 집계** → 별도 참조 열, **reach 분모에서 제외**.
- **지표 (기준 원장)**:

| 지표 | 정의 | 하중/튜닝 | 한계 |
|---|---|---|---|
| **시장 침투 폭 (reach)** | 한 곡이 진입한 **조사 국가 수** (GLOBAL 제외) | 무하중(카운트) | 조사 국가 집합에 상대적 — "세계 도달" 아니라 "조사한 N개국 중" |
| **지역 편중도** | 한 팀의 진입 국가를 권역별 집계 → 최다 권역 점유율(%) | 무하중(비율) | 권역 그룹핑(튜닝가능)에 종속 |
| **송곳 후보 임계** | (예) `reach ≥ T_reach` 또는 특정 권역 점유 ≥ `T_region` | **하중 · 값=도메인 소유자(A&R)** | 케이스 오버핏 금지 — 값은 결과 보기 **전에** 도메인 근거로 |
| **개척 시장 (proven market)** | `≥ market_min` 로스터 팀이 진입한 국가 = 검증된 K-pop 시장 | **하중 · 값=A&R** (`--market-min`, 기본 2) | 로스터·조사 국가 집합에 종속. market_min↓=노이즈, ↑=메가마켓만 |
| **화이트스페이스 (gap)** | 개척 시장 中 특정 팀이 **미진입**(rank=null)인 국가 = greenfield 후보 | 무하중(집합 차) | v1은 '미진입'만(약진입=고순위는 미포함, 후속 튜닝). **후보 신호이지 진출 지시 아님**(§0) |
| **신인 코호트 (rookie)** | 데뷔연도 ≥ (스냅샷연도 − `ROOKIE_YEARS`). 기본 `ROOKIE_YEARS=3` | **하중 · 값=A&R** (튜닝가능) | 데뷔연도 미해석 아티스트는 코호트 판정 제외(§4.2 커버리지 한계 승계). "신인=무베이스"의 근사 |

> **왜 코호트인가**: reach 정렬은 대형 팀을 전면화해 "대기업 아니면 불가능"으로 읽힐 위험. **신인 코호트를 표면화**하면 반대 신호가 드러난다 — 무베이스 신인도 특정 시장엔 닿는다(실측: CORTIS 2025→10개국, ILLIT 2024→6, Hearts2Hearts 2025→5). 제품 1차 사용자가 **신인 걸그룹**([`../../PRODUCT.md`](../../PRODUCT.md))이므로 이 렌즈가 핵심.

- **뷰 (대시보드 heatmap 계약 §4.1 재사용)**:
  - **최광역 랭킹**: 전 국가 union에서 `reach ≥ 2` 곡을 reach 내림차순 상위 `K`행(기본 15), cols = 권역순 국가, cells = 순위. **앵커 없음.**
  - **지리 지문**: 상위 reach 아티스트 × 권역순 국가, cells = 그 아티스트 곡의 국가별 **최고(min) 순위**. "이 팀은 어디에 사나".
  - **화이트스페이스**(기획, 로스터 스코프 전용) = **tunable 뷰**: 상위 아티스트 × **개척 시장만**(≥`market_min`팀), cells = 순위 / **빈칸 = 미개척 = greenfield 후보**. "닿을 수 있는데 아직 안 닿은 곳". 지문에서 강세 시장만 필터해 갭을 부각.
    - **tunable 계약**(범용, report-schema `chart.type:"tunable"`): report가 **전체 로스터×시장 행렬 + knob**을 실으면 대시보드가 슬라이더를 렌더하고 `market_min`을 **client-side로 재계산**(static-first 유지, 백엔드 0). `data = { view:"whitespace", matrix:{rows,cols,cells}, knobs:[{key,label,default,min,max,step}], topRows, note }`. **임계값=A&R가 직접 조정**(§2.1 "값=도메인 소유자"를 UI로 실현). 아무 모듈이나 이 계약으로 튜닝가능 뷰를 낼 수 있다. 스키마 추가는 승인+대시보드 동시 갱신(AGENTS §0) 완료.
- **결정성**: 정렬 tie-break은 **전순서**로 고정(입력 순서 무관) — 최광역 랭킹 `(reach desc, track_key asc)`; **최다 권역**은 `(국가수 desc, 그 권역 최고순위 asc, 권역 선언순)` — 동수면 *더 높이 오른* 권역(예: BTS 동아시아4·중남미4 동점 → LATAM 최고순위가 높아 중남미).
- **한계 필수 명시(insights)**: (a) reach는 **조사 국가 집합에 상대적**(세계 절대치 아님), (b) 매칭 = 문자열(§4.1 한계 승계), (c) **Spotify 단일 플랫폼** — 국내 코어(Melon/Circle) 미반영, "국내 vs 글로벌"의 국내축은 Spotify-KR로 **근사**(유료 ISRC 애그리게이터가 후속 과제), (d) **신호이지 타겟 평결 아님**(§5) — 기획 결정은 책임질 인간이.

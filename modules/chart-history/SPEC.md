## 한 줄

차트 스냅샷(Kworb)을 **사실 지표**로 정규화해 `report.json`으로 낸다. 히트 예측기가 아니라 **차트 신호 요약**.

## 입력 / 출력

- **입력**: 차트 스냅샷 HTML(들) 또는 **스냅샷 스토어 디렉터리**(v3.2) + (v3, 선택) 엔티티 맵 JSON. 스모크는 **픽스처**, 라이브는 `fetch`/`collect`/`enrich` 서브커맨드.
- **출력**: `output/report.json` (공유 규격 준수).

## 핵심 지표 (정의는 RULES.md)

1. **차트 진입 곡 / 고유 아티스트** — 스냅샷 폭
2. **1위 · 1위 일간 스트림(±delta)** — 히어로 KPI
3. **최장 차트인(Days) / 최고 상승(P+)** — 차트 수명·모멘텀 신호(단일 스냅샷 내장)
4. **Top 10 곡 · 일간 스트림** (bar)
5. **Top 아티스트 · 차트 내 일간 스트림 합** (bar)
6. **(v2) 국가 교차** — 교차 시장 수·전 시장 진입·최광역 곡 + 시장별 순위 heatmap
7. **(v2) 날짜 시계열** — 순위 추이 line + 기간 최고 상승 (스냅샷 축적 시)
8. **(v3) 아티스트 원산지** — 국내(KR) 비중 + 원산지 분포 bar (MusicBrainz 엔티티 조인)
9. **(v3.2) 크로스뷰 모멘텀** — 일간 vs 주간 순위차(상승세) bar + 상승세 곡 수·최고 모멘텀
10. **(v3.3) 크로스소스 positioning** — Spotify vs Apple 플랫폼별 순위 heatmap + 양대 진입·플랫폼 편중

## UX (대시보드 뷰)

- **단일 플랫폼 입력**: KPI 행(진입곡·1위·최장차트인·최고상승) → Top 10 스트림 바 → Top 아티스트 바 → (다국가) 지리 리프레임·화이트스페이스 tunable → 인사이트/추천.
- **멀티플랫폼 입력(v4.1 수평 병렬** — 어떤 플랫폼도 '본편' 아님, 표시 순서 youtube→spotify→apple = 도메인 소유자 판단): **플랫폼별 1위 KPI 나란히** + 차트 플랫폼·Spotify 렌즈 밖 → **홈 Top10 × 렌즈 히트맵**(3렌즈 합의/발산) → **렌즈 네이티브 수치 bar**(youtube 조회·spotify 스트림 — apple RSS는 무수치, 혼합·환산 없음) → 아티스트 × 플랫폼 히트맵(워치리스트 우선) → Spotify 렌즈 밖 표면화 bar → 지리 리프레임(spotify 서브그룹 = 최심 레일 기준, 위계 아님 명시) → 인사이트/추천.

## 완료 정의 (Definition of Done)

- [ ] 유효 스냅샷 입력 → **스키마 유효 `report.json`** 산출 (smoke 통과)
- [ ] 필수 지표(진입곡·1위·Top10) 존재, 스트림 값 ≥ 0
- [ ] 엔트리 0인 입력도 graceful (빈 차트 대신 `insights`에 명시)
- [ ] 결정성: 같은 입력 → 같은 지표(`generatedAt` 제외)
- [ ] `lint`(ruff)·`typecheck`(pyright) 통과

## 빌드 단계

- **v1** ✅: 단일 스냅샷 → KPI + Top10/Top아티스트 bar. 순위변동·차트수명은 스냅샷 내장 필드로.
- **v2** ✅: 다중 스냅샷 `analyze` — **국가 교차 heatmap**(KR/Global/US/JP 실데이터 → 지역 팬덤 신호), **날짜 축적 시 라인(시계열) 자동 전환** + 기간 모멘텀. `charts[].data` 규약은 RULES §4.1.
- **v3** ✅: **엔티티 마스터 조인(MusicBrainz, CC0)** — 아티스트→표준 엔티티(원산지·MBID), 정규화 키로 매칭 견고화, **아티스트 원산지 분포** 신호. `enrich`(라이브·캐시)/`analyze --entities`(오프라인).
- **v3.1** ✅(위키 보강): **Wikidata 폴백**(다국어 라벨·별칭 검색)으로 로마자 미해석 해소 — 해석 36→45/50, 출처 병기(MB·Wikidata).
- **v3.2** ✅: **헤더 기반 파서**(daily/weekly 등 컬럼 변형 견고) · **크로스뷰 모멘텀**(일간×주간 순위차 = 가속 신호) · **다일 축적 파이프라인**(`collect`→dated store, `analyze <dir>`→라인 시계열).
- **v3.3** ✅: **크로스소스 positioning**(Spotify×Apple KR) — 제목 기준 매칭으로 플랫폼별 순위 비교(스트리밍 vs 다운로드 편중). 파서 추가 견고화(암묵적 `</tr>` 종료).
- **v3.4** ✅: **엔티티 별칭 크로스언어 매칭** — MB 별칭(코르티스↔CORTIS)으로 크로스소스 아티스트를 언어-견고·콜리전 방지 매칭(하이브리드: 제목 매칭 + 엔티티 콜리전 가드, 커버리지 무회귀·엔티티 확인 11곡). **레코딩 MBID 매칭은 실측 결과 MB 커버리지 부재로 불발** → 제목-언어 갭(만찬가↔Bansanka)은 유료 ISRC 소스 과제(한계). 남은: 실데이터 다일 라인 + 잔여 5팀.
- **v4** ✅ (D-016 멀티플랫폼): 소스 3레일(Kworb Spotify + **Apple 공식 RSS**(`collect-apple`, 스토어 계약 변환) + Kworb YouTube) · 스토어 `chart/<platform>/<cc>/` · `analyze --latest`(leaf별 최신 = 라이브 일간 뷰) · `--watchlist`(캐노니컬+히트맵 우선 행) · **플랫폼 교차 증강**(아티스트×플랫폼 히트맵 + Spotify 렌즈 밖 표면화 — 지리 뷰는 spotify 서브그룹 유지) · daily_collect가 매일 라이브 스토어를 분석해 **탭 라이브화**. 실측: Spotify 렌즈 밖 24팀(RESCENE·H2H·KiiiKiii·화사·QWER·한로로 등).

### 배경 (전략)

- **왜 Kworb**: Circle Chart는 TDM 금지·JS 로드로 스크랩 배제(D-005). Kworb는 정적·저ToS·다국가 집계 프록시 → 차트 히스토리 레일의 합법 1차 소스.
- **포트폴리오 가치**: "핵심 흐름(CLI→report.json→대시보드)"을 처음 관통시키는 세로 슬라이스. 이후 다른 모듈이 같은 계약으로 복제된다.

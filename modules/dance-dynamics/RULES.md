# dance-dynamics · RULES (불변식 · 규칙)

> 지표를 **정확히** 정의해 AI/구현자가 즉흥적으로 만들지 못하게 한다. 이 정의를 어기는 산출은 금지.
> 값 범위 불변식은 [`TESTS.md`](TESTS.md)에서 검증한다.

## 1. 입력 · 전처리

- 포즈 추정: **MediaPipe Pose (33 landmarks)** 기본. landmark 신뢰도 < 0.5는 해당 프레임 보간 또는 제외.
- 프레임레이트를 고정 fps로 리샘플(기본 30fps)해 `dt`를 일정하게.
- **정규화 (필수 불변식)**: 모든 좌표는 **몸통 길이**(어깨 중점 ↔ 골반 중점 거리)로 나눠 스케일 정규화한다. → 카메라 거리·체격 차 보정. **이 단계 없으면 산출 무효.**

## 2. 다이내믹 계산 (정의)

정규화 좌표 `p(t)` 기준:

- **속도** `v(t) = |p(t) − p(t−1)| / dt` (관절별)
- **가속** `a(t) = Δv/dt`, **저크** `j(t) = Δa/dt`
- **에너지 엔벨로프** `E(t) = Σ_joint w_joint · |v_joint(t)|` — 관절 가중합 (가중치 `w`는 상수 테이블, 기본 균등)
- **액센트(accent)**: `E(t)`의 국소 최대점 중, **직후 짧은 구간(≤150ms) 내 급감**(저크 스파이크 동반) 조건을 만족하는 지점. "찍고 멈추는" 순간.

## 3. 지표 (정의 · 범위)

| 지표 | 정의 | 범위(불변식) |
|---|---|---|
| **다이내믹 레인지 (DR)** | `(p90(E) − p10(E)) / (p90(E) + ε)` | `[0, 1]` |
| **액센트–온셋 정합도** | 액센트 시각이 음악 온셋 ±100ms 윈도우 안에 드는 비율 | `[0, 100]` % |
| **평균 타이밍 오프셋** | 정합된 액센트의 (액센트시각 − 온셋시각) 평균 | ms (부호 = 빠름(−)/느림(+)) |
| **음악–움직임 상관** | `corr(리샘플 E(t), 음악 onset_strength/RMS(t))` | `[−1, 1]` |
| **바디파트 다이내믹** | 부위별 `Σ E`(정규화) → (시간 × 부위) 히트맵 | ≥ 0 |

- ε는 0으로 나눔 방지 상수. 음악 트랙: `librosa.onset.onset_strength`, `librosa.feature.rms`.

## 4. 출력 규칙 (report.json)

- `metrics`에 **반드시 포함**: 다이내믹 레인지, 액센트–온셋 정합도, 음악–움직임 상관.
- **음악 트랙이 없으면**: 음악 의존 지표(정합도·상관·오프셋)를 **생략**하고, 그 사실을 `insights`에 명시한다. (누락 금지 — 명시적 생략)
- 비교 모드: 원본 대비 커버의 DR/부위별 갭을 `insights`·히트맵에 포함.
- `generatedAt`은 실행 시각(ISO). 결정성 위해 **랜덤 시드 고정**.

## 5. 금지 (윤리 · 과대주장 — [`../../AGENTS.md`](../../AGENTS.md) §5)

- 점수를 "실력 총점 / 합격·불합격 / 데뷔감" 판정으로 표기 **금지**.
- 표현·감정·표정·비주얼·스타성을 정량화 **금지** (이 모듈은 다이내믹 축 하나만 측정).
- 2D 포즈의 깊이 손실 한계를 리포트에 **명시**.

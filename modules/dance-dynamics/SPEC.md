# dance-dynamics · SPEC (무엇을 만드나)

> 모듈 하네스의 "무엇을/UX/완료조건". 상세 규칙은 [`RULES.md`](RULES.md), 수용조건은 [`TESTS.md`](TESTS.md).
> 전략적 배경(Laban·North Star)은 이 문서 하단 참조. 도메인 근거: [`../../DOMAIN.md`](../../DOMAIN.md).

## 한 줄

안무 영상의 **다이내믹**(강-약 대비, 액센트 타이밍)을 정량화하고, 음악과의 정합을 측정해 **피드백**을 낸다. 정확도 채점기가 아니라 다이내믹 피드백 도구.

## 입력 / 출력

- **입력**: 안무 영상(mp4) + (선택) 음악 트랙. 비교 모드에선 원본 + 커버 2개.
- **출력**: `output/report.json` (공유 규격 준수) + 히트맵/타임라인 이미지.

## 핵심 지표 (정의는 RULES.md)

1. **다이내믹 레인지** — 강-약 대비폭 (히어로 KPI)
2. **액센트–온셋 정합도** — "찍는" 순간이 음악에 붙는가
3. **바디파트 다이내믹 히트맵** — 어디서 다이내믹이 나오나
4. **음악–움직임 다이내믹 상관** (North Star) — 몸이 음악의 다이내믹을 번역하는가

## UX (대시보드 뷰)

- 상단 KPI 행(위 지표) → 다이내믹 타임라인(+비트 그리드) → 바디파트 히트맵 → 인사이트/추천.
- 비교 모드: 원본 vs 커버 다이내믹 갭 히트맵.

## 완료 정의 (Definition of Done)

- [ ] 유효 영상 입력 → **스키마 유효 `report.json`** 산출 (smoke 통과)
- [ ] 다이내믹 레인지·정합도·상관이 정의된 범위 내 값
- [ ] 정규화로 카메라 거리/체격 차 보정 (TESTS의 정규화 검증 통과)
- [ ] 음악 없는 입력도 graceful (음악 의존 지표 생략 + insights에 명시)
- [ ] 결정성: 같은 입력 → 같은 출력

## 빌드 단계

- **v1**: 영상 1개 → 다이내믹 타임라인 + 다이내믹 레인지 1개.
- **v2**: 원본 vs 커버(DTW) + 바디파트 히트맵 + 자동 인사이트.
- **v3**: 음악–움직임 상관(North Star) + 리포트 완성형.

---

### 배경 (전략)

- **Thesis**: 춤의 본질은 정확도가 아니라 다이내믹. 칼군무 싱크는 결과, 다이내믹은 원인.
- **이론 앵커**: Laban Movement Analysis(Effort) — Time(급/지속)·Weight(강/약)·Flow(통제/해방).
- **North Star**: "좋은 댄서는 음악의 다이내믹을 몸으로 번역한다."
- **포트폴리오 가치**: "춤의 본질은 다이내믹"이라는 관점 자체가 taste의 증거. 안무(움직임)와 A&R(음악)을 잇는다.

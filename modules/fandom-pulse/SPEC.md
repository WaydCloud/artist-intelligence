> 근거·소스: [`../../DATA_SOURCES.md`](../../DATA_SOURCES.md)(Apify §1·소셜 §3·§4, D-002) · 윤리·바닥전제: [`../../AGENTS.md`](../../AGENTS.md) §2.1·§5 · [`../../DOMAIN.md`](../../DOMAIN.md) §0.

## 한 줄

공개 소셜(IG 해시태그) 게시물을 **facts-only 지표**로 축소해 해시태그/챌린지의 **화력·참여·모멘텀 신호**를 `report.json`으로 낸다. 인기·바이럴 **예측기가 아니라** 공개 반응 **신호 요약**.

## 입력 / 출력

- **입력**: facts-only 스냅샷 JSON(들) — 원문·PII 제거된 게시물 지표 레코드. 스모크는 **픽스처**, 라이브는 `fetch`(Apify)가 원본을 **즉시 축소**해 생성.
- **출력**: `output/report.json` (공유 규격 준수).

## 핵심 지표 (정의는 RULES.md §3 기준 원장)

1. **게시물 수 / 참여 총합** — 화력 폭
2. **중앙값 좋아요·댓글** — 대표 참여(평균 대신 중앙값: 바이럴 1건 왜곡 견고)
3. **고참여 게시물 수** — 상위 분위 임계값(튜닝) 초과 게시물
4. **릴스 비중(%)** — 콘텐츠 형식 믹스(챌린지/댄스 신호)
5. **(다일) 게시 가속** — 최근/이전 게시량 차 (모멘텀 신호)
6. **Top 공동 해시태그** (bar) — 확산/도달 신호
7. **일별 게시량** (line) — 게시 케이던스
8. **Top 사운드** (bar) — 트렌딩 사운드 신호(있을 때, DD 연결)

## UX (대시보드 뷰)

- 상단 KPI 행(게시물·참여·중앙값·릴스비중) → 공동 해시태그 bar → 일별 게시량 line → (사운드 bar) → 인사이트/추천.

## 완료 정의 (Definition of Done)

- [ ] 유효 facts-only 스냅샷 → **스키마 유효 `report.json`** (smoke 통과)
- [ ] 필수 지표(게시물 수·참여·중앙값) 존재, 모든 수치 ≥ 0
- [ ] 게시물 0인 입력도 graceful (빈 차트 대신 `insights` 명시)
- [ ] 결정성: 같은 입력 → 같은 지표(`generatedAt` 제외)
- [ ] **기준 원장**: 하중 기준(고참여·모멘텀 임계값)이 RULES §3에 원장 형식 + **튜닝 노출**(CLI 플래그, §2.1)
- [ ] fetch는 **facts-only만 저장**(PII·원문 미저장, §4)
- [ ] `lint`(ruff)·`typecheck`(pyright) 통과

## 빌드 단계

- **v1** (이번): 단일 해시태그 스냅샷 → 화력·참여·믹스 KPI + 공동해시태그 bar + 일별 게시량 line + 사운드 bar. `apify/instagram-hashtag-scraper` 레일(실측 검증 완료).
- (예정) **v2**: 다일 축적(`collect`)으로 실데이터 모멘텀 라인 · 프로필/릴스 소스 · 크로스플랫폼(TikTok 등 ToS 검토 후).

### 배경 (전략)

- **왜 IG 해시태그**: Apify 레일 D-002 3번, IG 공개(로그아웃)는 §4 sanctioned 경로. 실측 검증 완료(DATA_SOURCES "Apify 실측 검증"). **셀 수 있는 지표**라 주관 NLP 없이 §2.1 기준 원장을 깔끔히 실증.
- **포트폴리오 가치**: 검증된 데이터 레일 → 스키마 유효 report.json 관통(2번째 세로 슬라이스). 그리고 "기준을 명시·튜닝가능하게 구조화"하는 실력(책임소재 불변식·기준 원장)을 코드로 실증.

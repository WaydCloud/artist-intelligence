# signal-bridge · RULES (불변식 · 기준 원장)

> 브리지의 미시 판단(온셋·선행 분류 기준)을 **정확히** 정의해 즉흥 산출을 막는다. §3이 이 모듈의 **기준 원장**([`../../AGENTS.md`](../../AGENTS.md) §2.1). 값 무결성은 [`TESTS.md`](TESTS.md)에서 검증.

## 1. 입력 · 조인 규칙

- **입력 = signal-series 2개** (계약 정본: [`SPEC.md`](SPEC.md)). `--social`은 `signal=="social-buzz"`, `--chart`는 `signal=="chart-rank"`여야 하며 아니면 **실패**(우회 금지, §4). 네트워크 없음 — 수집은 소스 모듈 소유.
- **모듈 독립**(D-007/D-010): 브리지는 fandom-pulse·chart-history **코드를 import하지 않는다**. 공유는 **데이터**(signal-series JSON) — report-schema의 시간축 짝.
- **조인 키 = entity-master 최상위 캐노니컬 키**. 양 모듈이 동일 키에 수렴(SPEC 참조). 미매치 팀은 **원 라벨 유지** — 소셜-온리/차트-온리로 살아남는다(pre-mainstream 표면화, D-010).
- **결측 의미**: 소셜 값 `0` = 무신호, 차트 값 `null` = 차트 밖. 온셋은 각 시계열의 **자기 일자축**에서 첫 임계 통과일로 계산.

## 2. `charts[].data` 규약 (대시보드 렌더 계약)

- bar = `[{name, value}]` · line = `{x:[label], series:[{name, values:[num|null]}]}` (chart-history RULES §4.1·fandom-pulse RULES §3과 동일). **스키마 무변경**.
- **tunable(view=leadlag)** (v1.3): θ 슬라이더 — 원자료 시계열 + knobs를 실어 대시보드가 **클라이언트에서 온셋·분류를 재계산**(static-first·백엔드 없음). §2.1 완성형: 임계값을 A&R이 대시보드에서 직접 돌려본다(값=도메인 소유자). `data = { view:"leadlag", socialDates:[...], chartDates:[...], series:{key:{social?:[num], chart?:[num|null]}}, knobs:[{key,label,default,min,max,step}], note }`. 재계산 규칙은 §3 온셋 정의와 **동일해야 한다**(소셜 첫 `≥θ_social`·차트 첫 `≤θ_rank`·lead=차트−소셜) — 리포트 정본 수치는 CLI θ로 고정, 슬라이더는 탐색 뷰.

- **θ_rank 값 가이드**: 다일 store(전향)엔 `50`(상위 유의미 진입), 회고 역산엔 `200`(top-200 = "차트인 여부") 권장 — 시장 깊이·질문에 따라 A&R 조정.
- **정직성 마커 전파**: 차트 signal-series `provenance.synthetic == true`면 **메커니즘 시연 경고**, `reconstructed == true`면 **회고·역산 경고**(진입일 실제·중간 근사)를 최상단 insight로 방출. 합성/근사는 실증 강도가 다름을 명시(§0).

## 4. 출력 규칙 (report.json)

- `moduleId = "signal-bridge"`. 필수 상위 필드는 공유 스키마 준수([`../../packages/report-schema/report.schema.json`](../../packages/report-schema/report.schema.json)).
- `generatedAt` = 실행 시각(UTC ISO). 지표·차트·분류는 입력 signal-series에서 **결정적** 도출(시각 외 비결정 금지; `series` 키·일자 정렬).
- 입력이 비거나 조인 0이어도 크래시 금지 — 유효 report + insight 명시.
- `insights`에 **반드시 병기**: (a) 합성 입력이면 **시연 경고**(실증 아님), (b) **lead ≠ 인과**·표본 극소, (c) **반례(chart-led)** 노출, (d) 조인 방식(공유 캐노니컬), (e) 온셋 기준·임계값=A&R 소유.

### 4.1 프로필 라인 표시 규약 (D-013 · 대시보드 카드 렌더 계약)

> 프로필은 스키마상 **평문 insight**(스키마 무변경)이지만, 라인 문법을 고정해 대시보드가 **구조화 카드**로 렌더할 수 있게 한다. 파싱 실패 시 대시보드는 **평문 폴백**(규약은 표시 향상이지 계약 파괴 아님).

```
[프로필] <key> — <class>[(±Nd)] · 소셜 <N>건·참여 <E> · 드라이버: <why> · <차트요약>[ · YT 구독 <S>[·'<title>' +<V>/일]] → <활용 옵션>(§0)
```

- **구간 구분자**는 공백 포함 시퀀스 3종뿐: `" — "`(키/본문 1회) · `" · "`(구간) · `" → "`(활용 옵션, 마지막 1회). `<class>` ∈ {social-led, coincident, chart-led, social-only, chart-only, no-signal}.
- **자유텍스트**(키·드라이버·YT 제목)에 구분자 시퀀스가 등장하면 이미터가 **무공백형으로 접는다**(`_seg`: `" · "`→`"·"` 등) — 문법 보존은 이미터 책임, 방어 파싱은 대시보드 책임.
- `<차트요약>` = `차트 최고 #<R>·<M>시장(<CC,…>)[·<pf+pf…>]` | `차트 최고 #<R>·시장 미상[·<pf…>]` | `차트 미진입`. 플랫폼 병기(D-016)는 단일 spotify면 생략. 활용 옵션 내부의 `—`·`·`는 자유(추가 분해 안 함).

## 5. 금지 (윤리 · 과대주장 — [`../../AGENTS.md`](../../AGENTS.md) §5, 바닥전제 §0)

- 선행(lead)을 **"소셜이 차트를 만든다"·"뜰 팀 예측"으로 단정 금지**. → "이 축에서 이런 시간 순서 신호"로만(평결 = 결정 레이어 침범, §0).
- 합성 픽스처 결과를 **실증으로 등치 금지**(시연 경고 필수). 실증은 라이브 다일 collect.
- 소셜-온리 관측대상을 **"진출하라"는 지시로 등치 금지** — 조사 우선순위 후보일 뿐, 개별 검증 필요.
- 다루는 것은 **집계 아티스트 신호**. 팬 개인 데이터는 범위 아님(소스 모듈이 fetch 단계에서 폐기).
